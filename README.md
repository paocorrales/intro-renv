# Proyectos reproducibles con renv 

Ningún código es una burbuja aislada. Casi cualquier análisis de datos o código depende de una serie de paquetes externos, como ggplot2 para la representación gráfica, data.table para el procesamiento de datos, knitr para la generación de informes, etc.

Sin embargo, los paquetes que se mantienen de forma activa se actualizan para corregir errores, añadir o eliminar funciones, o cambiar su comportamiento. Estos cambios pueden llegar a alterar tu análisis o generar diferencias sutiles (o no tan sutiles) en los resultados.
En este taller aprenderemos a usar renv para encapsular las dependencias de tu código (qué paquetes usas y qué versiones) y poder recrearlo en el futuro. 

Este repositorio contiene el código fuente de https://paocorrales.github.io/intro-renv/
